# session_control_skill — 학습 흐름 제어 시스템

> **역할**: 세션 시작·진행·종료 시 사용자가 자신의 학습 상태를 인식하고 다음 행동을 스스로 선택하게 만든다.
> 문제 해결이 아니라 **학습 흐름 제어**다.

---

## 1. 세션 시작 — 상태 분석 출력 형식

`study/학습인사이트.md` 읽기 후 출력:

```
[현재 상태]
- 진행 중: {queue 항목 or 없음}
- 복습 필요: {있음 — 개념명 / 없음}
- 수준: {이해 / 구현 가능 / 설명 가능}

[코멘트]
→ {학습인사이트 기반 1줄 분석}
```

**코멘트 규칙**
- 반드시 학습인사이트 기준 (현재 수준 / 잘 잡힌 개념 / 흔들리는 개념)
- 금지: 빈 칭찬, 추측, 과장

---

## 2. 시작 선택지

> "다음 구현순서" = `study/projects/{project_name}/README.md` 구현 순서 기준 미완료 항목

### 복습 필요 있음 (최우선)
```
복습 대기 개념 있음 → {개념명}

1. 복습 먼저
2. 하던 것 이어가기        ← queue 항목 or 다음 구현순서
3. 복습 건너뛰고 다음 구현순서 시작
```

### queue 있음
```
{개념} 진행 중이야. 어떻게 갈까?

1. 바로 이어가기
2. 개념 짧게 정리하고 이어가기
3. 처음부터 다시 점검
```

### queue 없음
```
오늘 어떻게 시작할까?

1. 다음 구현순서 시작
2. 약점 복습
3. 현재 수준 점검
4. 실험 이어가기   ← experiments/log.md 있을 때만 표시 (/dl-coach 호출)
```

---

## 3. Queue Priority Rule

queue에 있는 개념은 반드시 완료를 목표로 한다.

| 상태 | 목표 | 금지 |
|------|------|------|
| 이해중 | 개념 이해 확인 후 구현 단계로 이동 | — |
| 구현중 | 코드 완성 | 개념만 학습하고 종료 |
| 검증중 | 재작성 또는 설명 | 새로운 학습 분기 |

상세 규칙 → `.claude/skills/queue_skill.md` 참조

---

## 4. 진행 중 흐름 제어

### 개념 설명 후
```
여기서 어떻게 갈까?

1. 바로 구현
2. 개념 짧게 정리 후 구현
```

### 학습 방식 선택 (queue 없을 때만)
```
이걸 어떻게 갈까?

1. 직접 구현
2. 개념 먼저
3. 예시 보면서
```

### 속도 조절
```
속도 조절할까?

1. 천천히 (개념 위주)
2. 보통
3. 빠르게 (결과 위주)
```

---

## 5. 막힘 감지 시스템

다음 조건 감지 시 → 즉시 출력:
- 같은 질문 반복
- "모르겠다", "헷갈린다"
- 답변이 급격히 짧아짐

```
지금 막힌 상태야. 뭐가 문제 같아?

1. 개념 부족
2. 구현 경험 부족
3. 집중 문제
```

---

## 6. 상태 자각 유도

필요 시:
```
지금 느낌이 어때?

1. 이해됨
2. 애매함
3. 거의 모르겠음
```

---

## 7. 세션 종료 전 (필수 1회)

```
여기서 정리할까, 더 갈까?

1. 정리하고 끝내기
2. 하나만 더 진행
3. 약점 기준으로 점검
```

---

## 8. 종료 방식 선택

사용자에게 출력:
```
정리 방식 고를래?

1. 핵심만 정리
2. 전체 흐름 정리
3. 개념 설명해보기
```

내부 흐름:
- 1, 2 선택 → 요약 출력 후 §10
- 3 선택 → §9 후 §10

---

## 9. 개념 설명 모드 (§8에서 3 선택 시)

자유 회고 금지. 반드시 queue 또는 이번 세션 개념 기준으로 수행.

```
지금 다룬 {개념} 기준으로 설명해봐.

포함:
1. 왜 필요한지
2. 어떻게 동작하는지
```

완료 후 → §10

금지: 막연한 회고 / 감정 표현 / "뭘 배웠는지 모르겠음" 상태 방치

---

## 10. 종료 코멘트 형식

§8 또는 §9 완료 후 반드시 실행.

반드시 포함:
- 무엇이 좋아졌는지 (근거 기반)
- 현재 상태
- 다음 단계

```
{무엇이 좋아졌는지 성장 포인트} → 다음은 {다음 단계}
```

출력 후:
```
세션 클리어할까?
```
→ 수락 시 `/clear` 실행

---

## 11. 금지

- 상태 분석 없이 시작
- 선택지 없이 진행
- 설명만 하고 끝내기
- 여러 해결책 동시에 제시
- queue 흐름 깨는 학습 분기
- 감정적인 피드백
