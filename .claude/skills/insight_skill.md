# insight_skill — 세션 종료 인사이트 처리 규칙

> §8 세션 종료 시 실행. document_skill과 독립. 별도 흐름 금지.

---

## 실행 순서

1. 학습인사이트.md 작성
2. 로그 파일 생성
3. 성장 지표 계산
4. 로그 README 업데이트

---

## 1. 학습인사이트.md 작성 규칙

경로: `study/학습인사이트.md`
방식: **덮어쓰기** (누적 금지)

```markdown
# 학습인사이트

_updated: YYYY-MM-DD_HH-MM

## 현재 수준
> {이해 단계 / 구현 가능 단계 / 설명 가능 단계} — 한 줄 설명

## 잘 잡힌 개념
- **개념명**: 근거 한 줄

## 아직 흔들리는 개념
- **개념명**: 흔들리는 이유 한 줄

## 다음 세션 포인트
{다음에 집중할 것 1개 — 구체적 행동으로}
```

### 수준 판단 기준

| 수준 | 기준 |
|------|------|
| 이해 단계 | 개념을 설명할 수 있지만 코드로 구현 못함 |
| 구현 가능 단계 | 힌트 있으면 코드 작성 가능 |
| 설명 가능 단계 | 코드를 보지 않고 처음부터 재작성 + 설명 가능 |

---

## 2. 로그 파일 생성 규칙

**파일명은 직접 짓지 않는다. 반드시 스크립트를 호출한다.**

```bash
python3 scripts/save_insight_log.py <<'EOF'
{로그 내용}
EOF
```

파일명(`YYYY-MM-DD_HH-MM.md`)은 스크립트가 현재 시각으로 결정한다.  
직접 Write 도구로 `학습인사이트_log/` 에 파일을 생성하는 것을 금지한다.

```markdown
# 학습 로그 — YYYY-MM-DD_HH-MM

## 수준
{이해 / 구현 가능 / 설명 가능}

## 이번 세션에서 다룬 것
- {개념/코드명}: {이해중|구현중|검증중|검증 완료 후 큐가 삭제되었다면 완료}

## 잘 잡힌 개념
- **개념명**: 근거

## 흔들리는 개념
- **개념명**: 이유

## 다음 세션 포인트
{1개}
```

## 3. 성장 지표 계산 규칙

### 지표 정의

| 지표 | 기준 |
|------|------|
| 구현 가능 개념 수 | 코드 없이 재작성 가능한 개념 수 |
| 설명 가능 개념 수 | 자신의 말로 타인에게 설명 가능한 개념 수 |

### 데이터 소스

- `study/projects/{project_name}/README.md` — 구현 체크 완료 항목
- `study/약점노트/README.md` — 레벨별 active 개념 수
- 현재 세션의 검증 결과

## 4. 로그 README 업데이트 규칙

경로: `study/학습인사이트_log/README.md`
방식: **덮어쓰기**
정렬: 최근 기록은 시간순 (같은 날짜 여러 세션 모두 표시)

```markdown
# 학습 성장 로그

_updated: YYYY-MM-DD_

## 현재 상태 요약
수준: {이해 / 구현 가능 / 설명 가능}
구현 가능 개념: N개 / 설명 가능 개념: N개

## 최근 기록
| 날짜·시각 | 수준 | 핵심 변화 |
|-----------|------|-----------|
| [[YYYY-MM-DD_HH-MM]] | ... | ... |

## 성장 비교 (7일 전 vs 현재)
> 7일 전 기록 없으면 가장 오래된 기록 기준

| 항목 | 이전 | 현재 |
|------|------|------|
| 수준 | ... | ... |
| 구현 가능 개념 | N개 | N개 |
| 설명 가능 개념 | N개 | N개 |

## 성장 포인트
- {이전 대비 발전한 것}

## 부족한 부분
- {아직 흔들리는 것}

## 한 줄 피드백
> {응원 메시지 — 구체적 근거 포함, 빈 칭찬 금지}
```

---

## 5. 7일 비교 방식

```
1. study/학습인사이트_log/ 내 파일 목록 확인
2. 오늘 날짜 - 7일 이내 파일 있음?
     Yes → 해당 파일 기준
     No  → 가장 오래된 파일 기준 (없으면 비교 생략)
3. 수준·구현 가능 개념 수·설명 가능 개념 수 비교
```

---

## 6. 응원 메시지 생성 기준

- 빈 칭찬("잘했어요!") 금지
- 구체적 근거 포함: 어떤 개념을 어떻게 다뤘는지
- 흔들리는 부분은 부정하지 않고 다음 방향으로 연결
- 1~2문장

예시:
> ConvTranspose2d를 역전파와 분리해서 이해한 게 핵심 전환점이었어. 다음엔 super_init까지 설명 가능한 수준으로 만들면 모델 정의 전체가 잡힌다.
