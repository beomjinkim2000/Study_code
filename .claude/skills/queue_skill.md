# queue_skill — session_queue 상태 관리 규칙

> session_queue는 할 일 목록이 아니라 **현재 학습 상태 흐름**을 추적하는 시스템이다.

---

## 등록 조건

다음 상황에서 반드시 추가:

1. 새로운 개념 등장
2. 코드 구현 시작
3. 개념 설명 완료 (구현 불필요한 경우)

```
형식: - [ ] {project_name}/개념명 (상태: 이해중|구현중|검증중)
```
> 완료는 상태가 아니라 이벤트 — queue에 기록하지 않는다

> queue에 없는 개념은 검증 대상이 아니다. document_skill은 queue 기반으로만 실행.

---

## 상태 정의

### 이해중
- 개념 처음 접함
- 아직 설명 불가 / 구조 불명확

### 구현중
- scaffold TODO 구현 중
- scaffold python 실행 OK 대기 중 포함

### 검증중
- 코드 또는 개념 1차 완료 상태
- 재현 테스트 대기 상태

### 검증 통과
- 상태가 아닌 이벤트
- 즉시 document_skill 실행 → queue에서 제거

---

## 상태 흐름

```
이해중 → 구현중 → 검증중 → [검증 통과] → 제거   ← 구현 필요한 개념
이해중 →          검증중 → [검증 통과] → 제거   ← 구현 불필요한 개념
```

---

## 상태 전이 규칙

### 이해중 → 구현중
- 해당 개념이 코드 구현과 연결됨
- 또는 사용자가 구현 시작

### 이해중 → 검증중
- 구현 필요 없는 개념
- 설명 + 이해 확인 완료

### 구현중 → 검증중
- scaffold `python NN_이름.py` → OK 통과
- 노트북 독립 재작성 대기 상태

### 검증중 → [검증 통과] → 제거
- 조건: 노트북에 코드 없이 독립 재작성 가능 + 개념 설명 가능
- 즉시: document_skill 실행 → queue에서 삭제

---

## 검증 규칙

검증은 반드시 세션에서 직접 수행한다.

### 코드 검증
- 코드 없이 다시 작성 요청
- 막힘 없이 작성 가능 → 통과

### 개념 검증
- 왜 필요한지 설명 요청
- 자기 말로 설명 가능 → 통과

### 실패 기준
- 힌트 필요
- 부분 기억
- 구조 설명 불가

→ 검증 실패 → 구현중 상태 유지

---

## 금지

- 검증 없이 document_skill 실행
- 상태 건너뛰기
- queue 없이 document_skill 실행
- 개념 질문 직후 구현중 자동 전환
